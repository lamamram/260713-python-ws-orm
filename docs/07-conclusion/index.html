Python Avancé : FastAPI + ORM

Formation Python Avancé : FastAPI + ORM — 5 jours.

Bienvenue

Cette formation vous accompagne dans la construction d'une API REST complète en Python, de la définition des routes jusqu'à la persistance des données en base relationnelle. Elle s'adresse à des développeurs Python déjà opérationnels qui souhaitent maîtriser deux outils devenus incontournables dans l'écosystème backend Python : FastAPI pour l'exposition d'une API web moderne et SQLAlchemy pour la couche d'accès aux données.

L'objectif final : être capable de concevoir, implémenter, sécuriser et déployer une API REST complète avec FastAPI, alimentée par une base de données relationnelle gérée via SQLAlchemy, avec des migrations pilotées par Alembic — le tout documenté automatiquement, testé et prêt pour la mise en production.

Tour de table

Avant de commencer, un tour de table permet d'ajuster le niveau et le rythme de la semaine :

  • Quel est votre contexte professionnel actuel (type de projets, stack technique) ?
  • Avez-vous déjà exposé ou consommé une API REST, même dans un autre langage ?
  • Avez-vous une expérience avec un framework web Python (Flask, Django, Pyramid) ?
  • Avez-vous déjà interagi avec une base de données relationnelle depuis Python (sqlite3, psycopg2, SQLAlchemy) ?
  • Avez-vous un projet personnel ou professionnel que vous aimeriez prototyper pendant la formation ?

Objectifs de la formation

À l'issue de la formation, chaque stagiaire est capable de :

  • Comprendre et appliquer l'architecture REST pour exposer des ressources via une API HTTP
  • Créer un projet FastAPI complet avec routage, validation Pydantic et documentation OpenAPI automatique
  • Gérer les paramètres de requête, les corps JSON et les réponses avec les codes HTTP appropriés
  • Implémenter une authentification par jeton JWT avec le système de dépendances de FastAPI
  • Écrire des gestionnaires asynchrones (async/await) adaptés aux opérations I/O-bound
  • Modéliser une base de données relationnelle avec SQLAlchemy ORM (modèles, colonnes, relations)
  • Écrire des requêtes ORM de base et avancées (filtres, jointures, agrégations, eager loading)
  • Gérer l'évolution du schéma de données avec Alembic
  • Intégrer SQLAlchemy dans FastAPI via le système de dépendances (session par requête)
  • Tester une API FastAPI avec pytest et TestClient
  • Conteneuriser et déployer une API FastAPI avec Docker

Public concerné et prérequis

Cette formation s'adresse à des développeurs Python souhaitant passer à une architecture orientée services ou consolider une pratique existante des API et des ORM.

Prérequis :

  • Avoir suivi la formation "Python : Initiation + Approfondissement" ou posséder les notions équivalentes : manipulation de classes et d'objets, exceptions, fichiers, modules
  • Notions de base en SQL (SELECT, INSERT, JOIN) — utiles mais rappelées en contexte lors du Chapitre 4
  • Aucune connaissance préalable de FastAPI, SQLAlchemy ou de tout autre framework web Python n'est requise

Programme de la semaine

La formation se déroule sur cinq jours, organisés en deux blocs : les trois premiers jours couvrent FastAPI de A à Z (de la découverte jusqu'au déploiement sécurisé), les deux derniers jours couvrent SQLAlchemy puis l'intégration des deux outils dans un projet complet.

Chapitre Contenu
Chapitre 1 Découverte de FastAPI et des Web Services
Chapitre 2 Implémenter des routes REST avec FastAPI
Chapitre 3 Sécurité, dépendances et déploiement FastAPI
Chapitre 4 SQLAlchemy — Modélisation et requêtes
Chapitre 5 SQLAlchemy avancé et intégration avec FastAPI

La semaine se termine par un projet fil rouge : une API de gestion de catalogue (utilisateurs, articles, commandes) entièrement persistée avec SQLAlchemy, protégée par JWT, documentée via Swagger et conteneurisée dans Docker.

Organisation pédagogique

Chaque chapitre alterne apports théoriques et ateliers pratiques (environ trois quarts du temps de formation). Deux formats d'exercices sont proposés : des Travaux Pratiques guidés pas-à-pas pour consolider chaque notion, et des exercices ouverts (format Pro) pour les stagiaires qui souhaitent se confronter à un énoncé sans fil directeur.

Chaque chapitre se conclut par un quiz de validation de dix questions. Une vérification concrète (état de l'API, requêtes HTTP fonctionnelles) clôt chaque journée.
